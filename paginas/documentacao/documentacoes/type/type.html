TypeScript: Documentação.

Introdução ao TypeScript

O TypeScript é uma extensão do JavaScript desenvolvida pela Microsoft, projetada para adicionar tipagem estática e recursos avançados ao ecossistema JavaScript. Ele é um superset do JavaScript, o que significa que todo código JavaScript válido também é válido em TypeScript. No entanto, o TypeScript traz benefícios adicionais, como:

  • Tipagem Estática: Permite definir tipos para variáveis, funções e objetos, ajudando a detectar erros durante o desenvolvimento.
  • Melhor Suporte a Ferramentas: Facilita a autocompletar, refatoração e navegação no código.
  • Compatibilidade com JavaScript: O código TypeScript é compilado para JavaScript puro, garantindo compatibilidade com qualquer ambiente que execute JavaScript.

Com o TypeScript, você pode escrever código mais seguro, escalável e fácil de manter, especialmente em projetos de grande porte.

Para que serve o TypeScript?

O TypeScript é amplamente utilizado para melhorar a qualidade e a produtividade no desenvolvimento de aplicações JavaScript. Aqui estão alguns dos principais benefícios:

  • Detecção de Erros em Tempo de Compilação: A tipagem estática ajuda a identificar erros antes mesmo de executar o código.
  • Código Mais Legível e Organizado: A definição de tipos e interfaces torna o código mais claro e fácil de entender.
  • Refatoração Segura: Com tipos bem definidos, é mais seguro fazer mudanças no código sem introduzir bugs.
  • Suporte a Recursos Modernos: O TypeScript permite usar funcionalidades do ES6+ (como classes, módulos e arrow functions) em ambientes que ainda não suportam essas versões do JavaScript.

Ele é especialmente popular em frameworks como Angular, React e Vue.js, além de ser uma excelente escolha para back-ends em Node.js.

Aplicação prática do TypeScript

O TypeScript pode ser aplicado em diversos cenários de desenvolvimento, incluindo:

  • Desenvolvimento Web: Criação de aplicações front-end e back-end com tipagem estática e organização de código.
  • Desenvolvimento Mobile: Uso com frameworks como React Native para criar aplicativos móveis tipados e robustos.
  • Desenvolvimento de Bibliotecas: Criação de pacotes tipados que facilitam a integração e o uso por outros desenvolvedores.
  • Jogos e Ferramentas: Implementação em engines como Babylon.js para desenvolvimento de jogos e ferramentas gráficas.

Em resumo, o TypeScript é uma ferramenta versátil que pode ser usada em praticamente qualquer projeto JavaScript.

Instalação

Para começar a usar o TypeScript, você precisa instalá-lo globalmente em sua máquina. Abaixo estão os passos para instalação: 

npm install -g typescript

Após a instalação, você pode verificar a versão instalada com o seguinte comando: 

tsc --version

Isso garantirá que o TypeScript esteja pronto para uso em seu ambiente de desenvolvimento.

Tipagem Estática

Uma das principais características do TypeScript é a tipagem estática. Isso significa que você pode definir explicitamente os tipos de variáveis, parâmetros de funções e valores de retorno. Veja um exemplo: 

let nome: string = "João";
let idade: number = 25;
let ativo: boolean = true;

Com a tipagem estática, o TypeScript ajuda a evitar erros comuns, como atribuir um valor do tipo errado a uma variável.

Inferência de Tipo

O TypeScript é inteligente o suficiente para inferir tipos automaticamente quando você não os declara explicitamente. Por exemplo: 

let cidade = "São Paulo"; // TypeScript infere que "cidade" é do tipo string

Isso torna o código mais limpo, mas ainda mantém a segurança de tipos.

Tipos Primitivos

O TypeScript suporta os tipos primitivos básicos do JavaScript, além de alguns tipos adicionais. Aqui estão os principais:

  • string: Representa valores de texto, como "Olá, mundo!".
  • number: Representa números inteiros ou decimais, como 42 ou 3.14.
  • boolean: Representa valores verdadeiros ou falsos (true ou false).
  • null e undefined: Representam a ausência de valor.
  • any: Permite que uma variável receba qualquer tipo de valor, desativando a checagem de tipos.
  • void: Indica que uma função não retorna nenhum valor.
  • never: Representa funções que nunca retornam, como funções que lançam exceções ou entram em loops infinitos.

Estruturas de Dados

O TypeScript oferece suporte a várias estruturas de dados, como arrays, tuplas e enums. Veja exemplos abaixo:

Arrays

let numeros: number[] = [1, 2, 3];
let strings: Array<string> = ["a", "b", "c"];

Tuplas

let tupla: [string, number] = ["João", 30];

Enums

enum Cores {
  Vermelho,
  Verde,
  Azul
}
let corFavorita: Cores = Cores.Verde;

Objetos e Interfaces

Em TypeScript, você pode definir a estrutura de objetos usando interfaces. Isso ajuda a garantir que os objetos sigam um formato específico.

Definição de Objetos

let pessoa: { nome: string; idade: number } = {
  nome: "Maria",
  idade: 28
};

Interfaces

interface Pessoa {
  nome: string;
  idade: number;
}
let usuario: Pessoa = { nome: "Carlos", idade: 35 };

Funções

Funções em TypeScript podem ter tipos definidos para parâmetros e valores de retorno. Isso ajuda a garantir que a função seja usada corretamente. 

function somar(a: number, b: number): number {
  return a + b;
}

Parâmetros Opcionais e Padrão

function saudacao(nome: string = "Visitante", idade?: number) {
  return `Olá, ${nome}!`;
}

Classes e POO

O TypeScript suporta programação orientada a objetos (POO) com classes, herança e modificadores de acesso. 

class Animal {
  nome: string;
  constructor(nome: string) {
    this.nome = nome;
  }
  falar(): void {
    console.log(`${this.nome} faz um som.`);
  }
}
class Cachorro extends Animal {
  latir(): void {
    console.log(`${this.nome} late!`);
  }
}
let dog = new Cachorro("Rex");
dog.falar();
dog.latir();

Generics

Generics permitem criar componentes reutilizáveis que funcionam com qualquer tipo de dado. 

function identidade<T>(valor: T): T {
  return valor;
}
let numero = identidade<number>(10);
let texto = identidade<string>("Oi");

Modificadores de Acesso

O TypeScript oferece modificadores de acesso para controlar a visibilidade de propriedades e métodos:

  • public: Acessível de qualquer lugar (padrão).
  • private: Acessível apenas dentro da classe.
  • protected: Acessível na classe e subclasses.
class Conta {
  private saldo: number = 0;
  depositar(valor: number) {
    this.saldo += valor;
  }
}

Conclusão

O TypeScript é uma ferramenta poderosa que melhora a produtividade e a qualidade do código JavaScript. Com tipagem estática, suporte a POO, interfaces e módulos, ele é ideal para projetos que exigem robustez, escalabilidade e facilidade de manutenção.

Seja para aplicações web, mobile ou back-end, o TypeScript é uma escolha excelente para desenvolvedores que desejam escrever código mais seguro e organizado. Adote o TypeScript e leve seus projetos para o próximo nível!
